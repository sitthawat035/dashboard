# MediaSourcing Skill (lookforward Edition)

## 🎯 Purpose
จัดหาภาพประกอบที่ช่วยเพิ่มความน่าเชื่อถือและอธิบายเนื้อหาเชิงเทคนิค (AI, Tech, Crypto) โดยเน้นภาพที่ดูเป็น 'ของจริง' และ 'มีคุณค่า' มากกว่าภาพสวยงามทั่วไป

## 🖼️ Visual Standard (lookforward Style)

### ✅ DO (ภาพที่ควรหา):
- **Technical Screenshots**: หน้าจอ Dashboard, Code Snippets, หรือหน้าเว็บจริงของโปรเจกต์
- **System Diagrams**: แผนผังการทำงาน, Infographics สรุปสถิติ
- **Real Use-case**: ภาพการใช้งานจริงในสภาพแวดล้อมจริง
- **Minimalist AI Generated**: หากต้องเจนภาพ ให้ใช้สไตล์ Tech Schematic หรือ Blueprint

### ❌ DON'T (ห้ามใช้เด็ดขาด):
- **Generic Stock Photos**: รูปคนยิ้มเกินจริง, รูปหุ่นยนต์จับมือกับคน (ดูปลอม)
- **Low-Quality Memes**: หากไม่จำเป็นห้ามใช้มีมที่ดูลดทอนความน่าเชื่อถือ
- **Irrelevant Images**: ภาพที่ไม่เกี่ยวข้องกับเนื้อหาเชิงเทคนิค

---

## 🛠️ Sourcing Process

### Step 1: Visual Identification
- วิเคราะห์จาก Content Draft ว่าประเด็นไหนที่ต้องการภาพประกอบเพื่อ "พิสูจน์" หรือ "อธิบาย" ให้เห็นภาพชัดขึ้น

### Step 2: Source Selection
- ค้นหาจาก Official Assets ของโปรเจกต์นั้น ๆ (เช่น GitHub Repo, Project Docs)
- ค้นหาภาพสถิติหรือ Chart จากแหล่งข้อมูล Research

### Step 3: Optimization
- ตรวจสอบความคมชัดและสัดส่วนที่เหมาะสมกับ Platform
- เขียน **Alt Text** หรือคำอธิบายภาพที่เสริมความเข้าใจเชิงเทคนิค

- [ ] ภาพนี้ดูน่าเชื่อถือและไม่ดูเป็นภาพ Stock ใช่หรือไม่?
